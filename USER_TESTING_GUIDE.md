# HiveMind 使用者體驗測試指南

## 🎯 測試目標

以普通使用者的角度，測試 HiveMind 的完整使用流程，包括：
- Web UI 登入
- 任務上傳
- 任務執行
- 結果查看

---

## 🚀 開始測試（3 步驟）

### 步驟 1: 啟動系統

```powershell
./start_with_ui.ps1
```

**預期結果**:
- ✅ 自動啟動所有後端服務
- ✅ 自動啟動 Web UI
- ✅ 自動打開瀏覽器到 http://localhost:3000

**等待時間**: 約 10-15 秒

---

### 步驟 2: 準備測試任務

```powershell
cd test_tasks
./package_tasks.ps1
```

**預期結果**:
- ✅ 建立 3 個測試任務 ZIP 檔案
  - `01_hello_world.zip` - 簡單測試
  - `02_math_compute.zip` - 計算密集
  - `03_text_processing.zip` - 文字處理

---

### 步驟 3: 開始測試

打開瀏覽器訪問: **http://localhost:3000**

---

## 📝 測試流程

### 測試 1: 登入系統

#### 操作步驟
1. 在登入頁面輸入：
   - **用戶名**: `testuser`
   - **密碼**: `testpass123`
2. 點擊「登入」按鈕

#### 預期結果
- ✅ 成功登入
- ✅ 跳轉到任務管理頁面
- ✅ 顯示用戶名和餘額

#### 可能問題
- ❌ 登入失敗：檢查 Master 服務是否正常運行
- ❌ 頁面無法載入：檢查 http://localhost:3000 是否可訪問

---

### 測試 2: 上傳簡單任務

#### 操作步驟
1. 點擊「上傳任務」或「新增任務」按鈕
2. 選擇檔案：`test_tasks/01_hello_world.zip`
3. 設定資源需求：
   - **CPU**: 2 核心
   - **記憶體**: 1 GB
4. 點擊「提交任務」

#### 預期結果
- ✅ 顯示「任務提交成功」訊息
- ✅ 獲得任務 ID（例如：`task-abc123`）
- ✅ 任務出現在任務列表中
- ✅ 任務狀態為「PENDING」或「RUNNING」

#### 觀察重點
- 任務提交是否順暢
- 錯誤訊息是否清楚
- UI 回應是否即時

---

### 測試 3: 查看任務執行

#### 操作步驟
1. 在任務列表中找到剛提交的任務
2. 點擊任務查看詳情
3. 等待任務執行完成（約 1-3 秒）

#### 預期結果
- ✅ 任務狀態變為「RUNNING」
- ✅ 顯示執行的 Worker ID
- ✅ 任務狀態最終變為「COMPLETED」
- ✅ 顯示執行時間

#### 觀察重點
- 狀態更新是否即時
- 執行時間是否合理
- UI 是否流暢

---

- ✅ 顯示資源使用統計：

## ❓ 常見問題

### Q: Web UI 無法打開
**A**: 檢查服務是否啟動
```powershell
# 檢查 Master UI 是否運行
netstat -ano | findstr "3000"

# 查看 Job 狀態
Get-Job
```

### Q: 登入失敗
**A**: 檢查 Master 服務
```powershell
# 測試 Master API
curl http://localhost:8082/health
```

### Q: 任務一直 PENDING
**A**: 檢查 Worker 是否註冊
```powershell
# 連接 Redis
docker exec -it redis-hivemind redis-cli
KEYS node:*
```

### Q: 任務執行失敗
**A**: 查看錯誤訊息
- 在 UI 中查看任務詳情
- 檢查「標準錯誤」輸出
- 查看後端日誌
