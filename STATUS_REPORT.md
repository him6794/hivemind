## 最近更改 (2026-03-25)

### 新增功能
1. **詳細 Redispatch 日誌**
   - 任務分配成功/失敗
   - 超時重分配詳情
   - 重試次數和原因追蹤

2. **Redis 任務存儲**
   - 從 SQLite 單點遷移到 Redis
   - 支持橫向擴展
   - 自動過期清理
   - 任務索引優化

3. **UI 分離**
   - Master UI (端口 3000) - 任務管理
   - Worker UI (端口 3001) - 節點管理
   - 獨立的開發和部署流程

### Bug 修復
1. **DISPATCHED 狀態卡住**
   - 原因: 計費結算邏輯持續更新任務時間戳
   - 修復: 只結算 RUNNING 任務
   - 影響: 任務現在能正常超時和重分配

2. **日誌錯誤訊息不具體**
   - 原因: 返回通用錯誤訊息
   - 修復: 返回具體的狀態和原因
   - 影響: 更容易診斷問題

## 已知問題

### 🟡 待解決
1. **前端環境變量配置**
   - 需要手動設置 VITE_API_BASE
   - 建議: 使用 `.env` 文件或構建時參數

2. **日誌文件大小**
   - 長期運行可能導致 nodepool.log 過大
   - 建議: 實現日誌輪轉機制

3. **Redis 高可用**
   - 目前為單實例
   - 建議: 使用 Redis Sentinel 或 Cluster

### 🟢 已知限制
1. **Worker 硬體檢測**
   - 依賴 Worker Control API
   - 當前為本地模式，需要 Worker 程式運行

2. **任務文件大小**
   - 無明確限制，建議 < 1GB
   - 大文件建議使用 HTTP/Magnet 而非直接上傳
