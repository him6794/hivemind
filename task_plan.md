# Full Test And Review Plan

## Goal

Run a broad, evidence-backed test pass and code review for Hivemind, covering:

- Mock or placeholder data left in product paths.
- Security vulnerabilities.
- Performance bottlenecks.
- Business logic errors.
- Actual executable tests and runtime checks, not only static analysis.
- Delegated review work using subagents where available.

## Scope

- Rust workspace: `hivemind-rs/`
- Protobuf contracts: `proto/`
- Frontends: `frontend/`, `frontend/master-ui/`, `frontend/worker-ui/`
- Packaging and deployment scripts: `scripts/`, Docker files, compose files
- Docs only where they describe expected runtime behavior or expose risks

## Current Assumptions

- Existing uncommitted work belongs to the user and must be preserved.
- Generated directories such as `target*`, `dist`, and `node_modules` are excluded from source review unless needed as build evidence.
- "Complete" means tested and reviewed as broadly as the local environment allows, with limitations stated explicitly.
- Docker integration is no longer blocked: Docker health, compose config/build, and the one-shot compose test gate passed this round.
- Some residual hardening remains open: remote artifact materialization and unmaintained `unic-*` warnings in the pinned executor Ruff/Ty dependency stack.
